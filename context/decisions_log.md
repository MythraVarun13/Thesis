|---|---|---|---|

| 2026-05-31 | Treat `_time` as UTC (timezone Z suffix) | Timestamp format is ISO 8601 with `Z` = UTC | Must convert to CET/CEST for German local-time analysis; document conversion, never do silently |

| 2026-06-01 | Use DuckDB as primary query engine for all raw 40 GB data | Pandas chunking is too slow at this scale; DuckDB reads CSV directly without loading into memory, 30x faster for aggregations | All scripts 06+ use DuckDB; notebooks can also call DuckDB for signal deep-dives |
| 2026-06-01 | Energy signal aggregation: MAX−MIN per hour (not SUM) for cumulative counters | `greal_E_*` signals are ever-increasing totals since meter started, not energy-per-20-seconds; SUM of running totals produces nonsense | Critical for HP COP calculation — applying wrong aggregation would corrupt all energy analysis |
| 2026-06-01 | Store all processed data as a single wide Parquet file (`data/processed/hourly.parquet`) | Columnar format; 40 GB at 20s → ~200 MB at 1hr; loads into pandas in seconds; all notebooks use one file | Eliminates need to re-query raw CSVs for every analysis; single source of truth for EDA |
| 2026-06-01 | MPC is the primary end goal — full analysis of ALL 233 signals required | User confirmed MPC is the most important algorithm; HP FDD is a stepping stone, not the destination | All notebooks must cover all subsystems; no signal left as "TBD"; system models feed directly into MPC state-space |
| 2026-06-01 | Store Parquet with UTC timestamps; convert to Europe/Berlin only for plotting | Keeping UTC internally prevents accidental drift; DST (CET/CEST) only matters for occupancy and daily profile charts | Document every UTC→local conversion explicitly; never convert silently |
| 2026-06-01 | Forward-fill setpoint columns after resampling (`V_real*` signals) | Setpoints are event-driven — only written when value changes; most hours have no row; ffill propagates last known setpoint | Without ffill, setpoints are NaN for most hours, unusable for MPC constraint tracking |
| 2026-06-01 | Phase toolchain installation: EDA now → MPC/forecasting later | Avoid installing heavy packages (cvxpy, torch, sktime) before EDA proves they are needed; keeps environment lean | Install prophet+streamlit when MVP demo phase begins; cvxpy+scikit-learn when MPC phase begins |

| 2026-06-01 | Use `WHERE TRY_CAST(_value AS DOUBLE) IS NOT NULL` not `TRIM(_value) != ''` | DuckDB auto-detects _value as BIGINT for integer files; TRIM() fails on non-VARCHAR types | Consistent across all SQL in scripts 06, 07 and notebooks |
